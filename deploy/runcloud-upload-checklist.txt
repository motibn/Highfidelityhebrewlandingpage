פריסה — k8now.com (RunCloud + GitHub)
=====================================

המסך שלך מראה: Web App "k8now", דומיין k8now.com, Public Path =
  /home/runcloud/webapps/k8now/dist

כלומר השרת מגיש את האתר מתוך תיקיית dist אחרי בילד.

---
א. מה שקורה מהריפו (אחרי push ל־main)
--------------------------------------
1) ב־RunCloud: נכנסים ל־Web Application "k8now" → טאב Git.
2) לוחצים Deploy / Pull לפי הממשק שלכם (כדי למשוך את main האחרון מהריפו motibn/Highfidelityhebrewlandingpage).
3) ודאו בהגדרות ה־Git של האפליקציה שנקבעו פקודות בסגנון:
   - התקנה: npm ci   (או npm install)
   - בילד:   npm run build
   אחרי הבילד חייב להופיע ב־dist/ לפחות: index.html, תיקייה thank-you/index.html, assets/, .htaccess
   (העתק ל־thank-you/ עוזר ל־NGINX סטטי בלי try_files)

אם אין סקריפט בילד בפאנל — הגדירו את זה בתמיכת RunCloud או במסך Git של האפליקציה.

---
ב. מה שאתה עושה באתר פעם אחת (ב־RunCloud)
------------------------------------------
אם מתקבל 404 על /thank-you ללא slash: נסו קודם
  https://k8now.com/thank-you/
לאחר פריסת הבילד החדש. אם עדיין 404:

1) Web Application k8now → NGINX Config
2) הוסיפו (או שלבו בקיים) את התוכן מקובץ הריפו:
   deploy/runcloud-nginx-spa.conf
   — הבלוק עם:  try_files $uri $uri/ /index.html;
3) שמרו ועדכנו/רעננו Nginx.

---
ג. Origami (מחוץ לאתר)
----------------------
בממשק Origami, אחרי שליחת טופס: הפניה לכתובת עם slash בסוף:
  https://www.k8now.com/thank-you/
(או k8now.com ללא www — לפי הדומיין שלכם)
